Mad Libs Game















Addendum:

In this project, I created a web server using Express.js to host a madlib game. Here's an overview of my process and choices: 1. Server Setup: I chose Express.js for its simplicity and robust feature set. I used middleware for parsing form data, logging requests, and serving static files. 2. File Structure: I organized my static files (HTML, CSS) in a 'public' folder for clear separation from server-side code. 3. Routing: I implemented two main routes - a GET route to serve the form and a POST route to handle form submission. The POST route checks for complete form data and generates the madlib story. 4. Error Handling: I added checks to ensure all form fields are filled before generating the story, providing user feedback if fields are missing. 5. Port Selection: The server uses port 80 by default, but switches to 8085 if run with a 'local' argument, allowing for flexible deployment. 6. CSS Integration: I linked the CSS file in the HTML and served it as a static file. Initially, I faced a 404 error which I resolved by correcting the file path in the server configuration. 7. Learning Outcomes: This project taught me about server-side routing, form handling, and the importance of proper file paths in web development. I learned to use developer tools to debug issues like the CSS 404 error.
